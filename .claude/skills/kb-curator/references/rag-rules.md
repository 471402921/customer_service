# RAG 写作规范执行清单

写入或修改任何 Q&A 后,逐条对照检查。**这是七陌检索质量的硬约束**。

## 切片层

- [ ] **段落 ≤ 2048 字符**(七陌 chunk 上限)。超长按子主题拆分到下一个 `###`
- [ ] **段落自包含**:这段话从全文切出来后,读者不依赖上下文就能完整理解
- [ ] **不用指代词**:不写"它/该功能/上述/这个",直接重复关键概念名
- [ ] **答案前置**:每个 Q&A 答案的第一句就给结论,后面才解释/展开
- [ ] **关键词前置**:`### Q. xxx?` 的问句把核心关键词写在前半段(用户搜索时输入的就是这些词)

## 结构层

- [ ] **markdown 标题**:`#` `##` `###`,**不手写**"1.1""3.2"等数字编号
- [ ] **列表表达层级**:用 `-` `1.` 表达枚举,**不在文字里手写**"第一""第二"
- [ ] **Q&A 形式保留**:`### Q. <问句>?` 后接答案,不要用其他形式
- [ ] **每个 Q&A 独立**:即便话题相邻,也分开成两个 `### Q.`,不要塞到一个块里

## 内容层

- [ ] **URL 一字不改**:`https://platform.minimaxi.com/docs/...` 这种链接完整保留,不缩短不省略
- [ ] **代码示例完整**:JSON/cURL 示例不要为了短而砍参数
- [ ] **图片转文字**:如果源文档里是截图,改写成自然语言描述(七陌不解析图片)
- [ ] **表格简化**:复杂合并单元格的表改写为自然语言;保留的表保证每行数据独立完整

## 术语层(MiniMax 高频易错)

| 概念 | 正确表达 | 易错 |
|---|---|---|
| 国内 API 域名 | `minimaxi.com`(多个 i) | `minimax.com`(漏 i)|
| 国际 API 域名 | `minimax.io` | `minimax.com` |
| 国内/国际 base URL | OpenAI 兼容 `/v1`,Anthropic 兼容 `/anthropic` | 混用路径 |
| 海外平台登录 | 仅支持谷歌邮箱 | "邮箱" |
| 付费模式 | Token Plan / 按量计费 / 资源包 | 套餐(歧义) |
| 极速版 | highspeed | "高速版"/"快速版" |
| MiniMax Agent vs 开放平台 | 是两条独立产品线,术语不互换 | 混提"MiniMax Agent API" |
