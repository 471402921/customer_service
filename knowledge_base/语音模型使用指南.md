# 语音模型使用指南

## 语音模型选型

### Q. MiniMax 语音模型有哪些版本，分别适合什么需求?

MiniMax 语音模型(Speech 系列)目前提供 6 个可选版本，覆盖高保真(HD)与低延迟(Turbo)两条技术路线，开发者可根据音质要求与响应速度要求选择。

- **Speech-2.8-HD**(HD):精准还原真实语气细节，全面提升音色相似度，适合对真人化与还原度要求极高的场景。
- **Speech-2.8-Turbo**(Turbo):极速响应，语气表达生动自然，兼顾低延迟与表现力。
- **Speech-2.6-HD**(HD):极致音质与韵律表现，生成更快更自然，适合高质量长音频合成。
- **Speech-2.6-Turbo**(Turbo):音质优异，超低时延,响应更灵敏，适合实时交互。
- **Speech-02-HD**(HD):出色的韵律和稳定性，复刻相似度和音质表现突出，适合稳定的复刻场景。
- **Speech-02-Turbo**(Turbo):小语种能力增强，性能表现出色，适合多语种快速合成。

选型建议:实时对话、语音助手、客服机器人类场景优先选 Turbo;有声书、配音、音色复刻试听等离线或高保真场景优先选 HD。

## 同步语音合成(实时与低延迟场景)

### Q. 同步语音合成适合哪些场景，如何调用?

同步语音合成(MiniMax t2a_v2)适用于实时或低延迟场景，例如语音助手、实时播报、对话式产品的即时语音回复，调用后即时返回合成音频。它与异步语音合成是两个独立接口:同步用于即时返回，异步用于长文本批处理。

参考文档:

- 接口文档(HTTP):https://platform.minimaxi.com/docs/api-reference/speech-t2a-http
- 接口文档(WebSocket):https://platform.minimaxi.com/docs/api-reference/speech-t2a-websocket
- 使用指南:https://platform.minimaxi.com/docs/guides/speech-t2a-websocket

同步语音合成的核心能力与限制:

- 单次最长处理 **10,000 字符**
- 同时支持 HTTP 与 WebSocket 两种协议
- WebSocket 支持流式返回(适合边合成边播放)
- 支持音量、语调、语速、混音等控制
- 支持比特率、采样率等音频参数设置
- 返回音频时长、音频大小等元信息

同步语音合成 API 端点:

- HTTP:`POST https://api.minimaxi.com/v1/t2a_v2`
- WebSocket:`wss://api.minimaxi.com/ws/v1/t2a_v2`

### Q. 同步语音合成的关键参数有哪些?

同步语音合成调用 `t2a_v2` 接口时的关键参数说明如下，必填项决定能否成功生成，选填项控制音频效果与格式。

- **model**(必填):模型名称，例如 `speech-2.8-hd`，需选择上文列出的 Speech 系列模型版本。
- **text**(必填):待合成文本，最长 10,000 字符。
- **voice_setting.voice_id**(必填):音色 ID，可使用系统音色或自定义音色(快速复刻或音色设计生成的 voice_id)。
- **voice_setting.speed**(选填):语速，范围 0.5-2.0，默认 1.0。
- **voice_setting.vol**(选填):音量，范围 0-10，默认 1。
- **voice_setting.pitch**(选填):语调，范围 -12 到 12，默认 0。
- **audio_setting.sample_rate**(选填):采样率，默认 32000。
- **audio_setting.bitrate**(选填):比特率，默认 128000。
- **audio_setting.format**(选填):音频格式，可选 mp3(默认)、pcm、flac。

## 异步语音合成(长文本批处理场景)

### Q. 异步语音合成怎么用，与同步语音合成有什么区别?

异步语音合成(MiniMax speech-t2a-async)适用于长文本场景，例如有声书、长文播报、批量配音任务，需要分步创建任务、查询任务、下载结果。它与同步语音合成在适用场景、调用方式、文本长度、输出形式上存在显著差异。

参考文档:

- 接口文档(创建任务):https://platform.minimaxi.com/docs/api-reference/speech-t2a-async-create
- 接口文档(查询状态):https://platform.minimaxi.com/docs/api-reference/speech-t2a-async-query
- 使用指南:https://platform.minimaxi.com/docs/guides/speech-t2a-async

同步语音合成与异步语音合成的差异对照:

- **适用场景**:同步语音合成用于实时或低延迟需求(对话、播报、客服回复);异步语音合成用于长文本批量任务(有声书、长文配音、批量内容生成)。
- **调用方式**:同步语音合成通过 HTTP 或 WebSocket 即时返回音频;异步语音合成需依次执行"创建任务 → 轮询任务状态 → 下载音频"三个步骤。
- **文本长度上限**:同步语音合成单次最长 10,000 字符;异步语音合成单个文件长度限制为 **小于 10 万字符**。
- **输入方式**:同步语音合成只接受文本字符串;异步语音合成支持传字符串或先上传文本文件后引用 file_id。
- **输出能力**:异步语音合成额外支持返回句级时间戳(可用于自动生成字幕);同步语音合成不返回时间戳。
- **音频获取**:异步语音合成返回下载 URL，**有效期 9 小时**，需在有效期内下载;同步语音合成直接返回音频流或音频内容。
- **计费**:同步与异步语音合成均按合成字符数计费，计费规则一致(具体单价见《产品介绍与定价》)。

### Q. 异步语音合成的标准调用流程是什么?

异步语音合成的标准流程为四步，前一步成功后再进入下一步。

1. (可选)调用文件上传 API 将文本文件上传至平台，获取 file_id;若直接传字符串可跳过此步。
2. 调用创建语音生成任务 API，提交合成请求并获取 `task_id`。
3. 调用查询任务状态 API，使用 `task_id` 轮询，直到任务完成并拿到合成结果的 `file_id`。
4. 使用 `file_id` 调用文件下载 API 获取音频文件，注意下载 URL 有效期为 9 小时，需及时下载。

## 音色快速复刻(基于真实音频克隆)

### Q. 音色快速复刻怎么做，有哪些限制?

音色快速复刻(Voice Cloning)是指上传一段真人音频样本，由系统克隆出该说话人的音色，生成自定义 voice_id 用于后续语音合成。它与"音色设计"是两个独立功能:**音色快速复刻基于真实音频样本克隆音色，必须上传参考音频;音色设计则通过文本 prompt 直接生成全新音色，无需任何音频**。两者术语易混，请在选型时严格区分。

参考文档:

- 接口文档:https://platform.minimaxi.com/docs/api-reference/voice-cloning-clone
- 使用指南:https://platform.minimaxi.com/docs/guides/speech-voice-clone

音色快速复刻的标准步骤与限制:

1. 上传待克隆音频:
   - 格式:mp3、m4a、wav
   - 时长:10 秒 ~ 5 分钟
   - 大小:不超过 20MB
2. (可选)上传示例音频以增强复刻相似度和稳定性:
   - 时长:小于 8 秒
3. 调用复刻接口，基于 `file_id` + 自定义 `voice_id` 克隆音色。
4. 在语音合成接口的 `voice_setting.voice_id` 参数中使用该 `voice_id`。

重要时效约束:如果复刻音色 **7 天内未正式调用**，系统会自动删除该音色，需重新复刻。

### Q. 音色快速复刻如何计费?

音色快速复刻按音色个数计费，与音色设计单价相同但属于独立功能、独立扣费，互不冲抵。

- 音色复刻费用:**9.9 元/音色**
- 计费时点:在首次使用该复刻音色进行语音合成时收取，仅复刻不调用合成则不计费。
- 合成本身的字符费用单独计算，与复刻费用不冲抵。

## 音色设计(基于文本 prompt 生成全新音色)

### Q. 音色设计如何使用和计费?

音色设计(Voice Design)是指通过文本 prompt 描述目标音色的特征(如性别、年龄、风格等)，由系统直接生成全新的 `voice_id`，**无需上传任何参考音频**。这是与音色快速复刻最核心的差异:复刻依赖真实音频样本克隆已有说话人音色，设计仅依赖文字描述创造全新音色。

参考文档:

- 接口文档:https://platform.minimaxi.com/docs/api-reference/voice-design-design

生成的 `voice_id` 可直接在语音合成接口中使用。

音色设计的计费规则(独立计费，与音色快速复刻互不冲抵):

- 音色设计费用:**9.9 元/音色**
- 计费时点:在首次使用该设计音色进行语音合成时收取，仅设计不调用合成则不计费。
- 试听语音合成另行计费:按 **2 元/万字符** 收取试听费用。
- 合成本身的字符费用单独计算，与设计费用不冲抵。

## 音色管理

### Q. 如何管理和维护已有音色?

MiniMax 开放平台提供音色查询、删除接口，以及系统预置音色列表，便于统一管理自定义音色与查阅可直接使用的预置音色。

- 查询账户下可用音色 ID:https://platform.minimaxi.com/docs/api-reference/voice-management-get
- 删除音色:https://platform.minimaxi.com/docs/api-reference/voice-management-delete
- 系统音色列表(预置 voice_id):https://platform.minimaxi.com/docs/faq/system-voice-id

使用建议:开始集成前先查阅"系统音色列表"，若内置音色满足需求可直接引用对应 voice_id，无需额外付费;仅在系统音色无法满足业务调性时再使用音色快速复刻或音色设计。
