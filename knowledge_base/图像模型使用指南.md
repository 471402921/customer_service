# MiniMax 图像模型使用指南

## MiniMax 图像模型有哪些，分别适合哪些场景？

MiniMax 提供两款图像生成模型：image-01 和 image-01-live。

**image-01**

- 特点：画面表现细腻，写实感强，光影层次丰富，适合商业级图像需求
- 支持：文生图（text-to-image）、图生图（image-to-image）
- 适用场景：电商商品图、写实人像、复杂场景、专业创作

**image-01-live**

- 特点：手绘、卡通、插画等风格表现增强，支持画风参数快速调整
- 支持：仅文生图（text-to-image），不支持图生图
- 适用场景：动漫插画、手绘风、卡通风格内容创作

## 文生图接口如何调用，关键参数怎么选？

接口文档：https://platform.minimaxi.com/docs/api-reference/image-generation-t2i

使用指南：https://platform.minimaxi.com/docs/guides/image-generation

请求端点：`POST https://api.minimaxi.com/v1/image_generation`

**关键请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| model | string | 是 | 模型名称，可选 `image-01`、`image-01-live` |
| prompt | string | 是 | 图像描述文本 |
| aspect_ratio | string | 否 | 图像比例，可选 `1:1`（默认）、`16:9`、`4:3`、`3:2`、`2:3`、`3:4`、`9:16`、`21:9` |
| n | integer | 否 | 生成数量，范围 1–9，默认 1 |
| prompt_optimizer | boolean | 否 | 是否自动优化提示词，默认 `true` |
| response_format | string | 否 | 返回格式，可选 `url`（默认）或 `base64_image` |

**选择建议：**
- 社交媒体竖版内容：`aspect_ratio` 选 `9:16`
- 横幅、封面：`aspect_ratio` 选 `16:9`
- 头像、方形：`aspect_ratio` 选 `1:1`
- 需要批量选品时：`n` 最多可设为 9，一次请求生成多张

## image-01-live 的画风参数怎么配置？

image-01-live 支持 `style` 画风参数，可指定风格 ID 快速调整图像画风，实现风格一致性。

通过在请求 payload 中添加 `style` 字段传入，格式示例：

```python
payload = {
    "model": "image-01-live",
    "prompt": "一个女孩站在樱花树下",
    "aspect_ratio": "9:16",
    "style": "anime"   # 指定画风
}
```

**常见画风类型（以实际 API 文档支持值为准）：**

- `anime`：日式动漫风格
- `illustration`：插画风格
- `sketch`：手绘铅笔草图
- `watercolor`：水彩画风

> 具体可用的 `style` 枚举值及说明请参考官方接口文档：https://platform.minimaxi.com/docs/api-reference/image-generation-t2i

## MiniMax 文生图调用示例是什么？

```python
import requests, os

url = "https://api.minimaxi.com/v1/image_generation"
headers = {"Authorization": f"Bearer {os.environ.get('MINIMAX_API_KEY')}"}

payload = {
    "model": "image-01",
    "prompt": "描述文本",
    "aspect_ratio": "16:9",
    "response_format": "url",
    "n": 1
}

response = requests.post(url, headers=headers, json=payload)
data = response.json()
image_urls = data["data"]["image_urls"]
```

**响应结构：**

```json
{
  "data": {
    "image_urls": [
      "https://cdn.example.com/generated-image.jpg"
    ]
  }
}
```

## 图生图怎么生成"保留人物特征"的新图？

接口文档：https://platform.minimaxi.com/docs/api-reference/image-generation-i2i

图生图模式（仅 image-01 支持）可传入人物主体参考图，再结合 prompt 生成新图，在构图和场景变化下保持主体一致性。

**使用建议：**
- 建议上传**单人正面照片**以获得更稳定的效果
- 参考图支持 URL 或 base64 格式

**参数说明：**

| 参数 | 类型 | 说明 |
|------|------|------|
| model | string | 使用 `image-01`（image-01-live 不支持图生图） |
| prompt | string | 描述希望生成的新场景 |
| subject_reference | array | 参考图数组，包含 type 和 image_file 字段 |
| subject_reference[].type | string | 固定填 `character`（人物参考） |
| subject_reference[].image_file | string | 参考图 URL 或 base64 |
| aspect_ratio | string | 生成图的比例 |

## 图生图的请求示例是什么？

```python
import requests, os

url = "https://api.minimaxi.com/v1/image_generation"
headers = {"Authorization": f"Bearer {os.environ.get('MINIMAX_API_KEY')}"}

payload = {
    "model": "image-01",
    "prompt": "女孩在图书馆的窗户前，看向远方",
    "aspect_ratio": "16:9",
    "subject_reference": [
        {
            "type": "character",
            "image_file": "https://example.com/reference.jpg"
        }
    ],
    "response_format": "url"
}

response = requests.post(url, headers=headers, json=payload)
image_urls = response.json()["data"]["image_urls"]
```

## MiniMax 图像模型如何计费？

图像模型按**每张图片**计费，无论分辨率、比例、使用哪个模型，价格统一。

| 模型 | 计费单价 |
|------|----------|
| image-01 | 0.025 元/张 |
| image-01-live | 0.025 元/张 |

**计费示例：**
- 一次请求 `n=4`，生成 4 张图：0.025 × 4 = 0.1 元
- 批量生成 100 张图：0.025 × 100 = 2.5 元

按量计费定价参考：https://platform.minimaxi.com/docs/guides/pricing

## 图像模型速率限制是多少？

| 用户类型 | RPM（每分钟请求数） | TPM（每分钟 token 数） |
|----------|--------------------|-----------------------|
| 免费用户 | 10 | 60 |
| 充值用户 | 10 | 60 |

**注意：** 图像模型速率限制免费用户和充值用户相同，均为 RPM 10、TPM 60。

如需提升速率限制，可联系官方客服，或发送邮件至 api@minimaxi.com。
