# 图像模型使用指南

本文档介绍 MiniMax 开放平台图像模型的能力矩阵、文生图调用、image-01-live 画风参数、图生图调用以及计费规则。所有接口请求端点为国内域名 `https://api.minimaxi.com`(注意域名带额外的 i),Authorization 使用 Bearer Token 认证。

## MiniMax 图像模型矩阵

### Q. MiniMax 图像模型有哪些?分别适合哪些场景?

MiniMax 开放平台目前提供两款图像模型,分别覆盖通用图像生成与画风增强场景。

- **image-01**:支持文生图、图生图,画面表现细腻,支持自定义宽高(width / height 像素尺寸)。适合需要精细构图、自定义尺寸或需要保留人物主体特征的图生图场景。
- **image-01-live**:支持文生图,在手绘、卡通等画风上有专门增强,并支持画风参数(style.style_type / style.style_weight)。适合漫画、元气、中世纪、水彩等风格化创作场景。

## MiniMax 文生图(text-to-image)接口

### Q. MiniMax 文生图接口如何调用?关键参数怎么选?

MiniMax 文生图接口用于根据文本 prompt 生成图像,支持 image-01 与 image-01-live 两款模型。

- 接口文档:https://platform.minimaxi.com/docs/api-reference/image-generation-t2i
- 使用指南:https://platform.minimaxi.com/docs/guides/image-generation
- 请求端点:`POST https://api.minimaxi.com/v1/image_generation`

MiniMax 文生图接口关键参数:

- **model**(必填):取值 `image-01` 或 `image-01-live`。
- **prompt**(必填):图像文本描述,最长 1500 字符。
- **aspect_ratio**(选填):宽高比,默认 `1:1`。可选值 `1:1` / `16:9` / `4:3` / `3:2` / `2:3` / `3:4` / `9:16` / `21:9`,其中 `21:9` 仅 image-01 支持。
- **width / height**(选填):自定义像素尺寸,仅 image-01 支持,取值范围 [512, 2048],且必须为 8 的倍数。
- **response_format**(选填):返回格式,`url`(默认,链接 24 小时有效)或 `base64`。
- **n**(选填):单次生成图像数量,范围 [1, 9],默认 1。
- **prompt_optimizer**(选填):是否自动优化 prompt,默认 `false`。
- **seed**(选填):随机种子,用于复现同一张生成结果。

### Q. MiniMax 文生图调用示例(Python)是什么?

下面是 MiniMax 文生图接口的 Python 调用示例,使用 image-01 模型生成 16:9 比例的单张图像,返回 URL 链接。

```python
import requests, os

url = "https://api.minimaxi.com/v1/image_generation"
headers = {"Authorization": f"Bearer {os.environ.get('MINIMAX_API_KEY')}"}

payload = {
    "model": "image-01",
    "prompt": "描述文本",
    "aspect_ratio": "16:9",
    "response_format": "url",
    "n": 1
}

response = requests.post(url, headers=headers, json=payload)
```

请求需在 headers 中携带 `Authorization: Bearer <MINIMAX_API_KEY>`,POST 至 `https://api.minimaxi.com/v1/image_generation`。

## image-01-live 画风(style)参数

### Q. MiniMax image-01-live 的画风参数怎么配置?

MiniMax image-01-live 模型支持画风设置,可用于快速调整生成图像的风格一致性,适合手绘、卡通等风格化场景。画风参数通过请求体中的 `style` 对象传入,仅 image-01-live 支持,image-01 不支持该参数。

image-01-live 画风参数:

- **style.style_type**:画风类型,可选 `漫画` / `元气` / `中世纪` / `水彩` 四种。
- **style.style_weight**:画风权重,取值范围 `(0, 1]`,默认 `0.8`。数值越大,画风特征越明显。

## MiniMax 图生图(image-to-image)接口

### Q. MiniMax 图生图怎么生成"保留人物特征"的新图?

MiniMax 图生图模式可传入人物主体参考图,再结合 prompt 生成新图,在构图和场景变化下保持主体特征一致性,适合人物 IP 在不同场景下的复用。图生图与文生图共用同一接口端点,通过传入 `subject_reference` 参数切换为图生图模式。

- 接口文档:https://platform.minimaxi.com/docs/api-reference/image-generation-i2i
- 请求端点:`POST https://api.minimaxi.com/v1/image_generation`(使用 image-01 模型,并传入 `subject_reference`)

MiniMax 图生图关键参数:

- **subject_reference**(必填):人物主体参考数组。
- **subject_reference.type**(必填):主体类型,目前仅支持 `character`(人像)。
- **subject_reference.image_file**(必填):参考图 URL 或 Base64,支持 JPG / JPEG / PNG 格式,文件大小 ≤ 10MB。

效果建议:上传**单人正面照片**作为参考图,可获得更稳定的人物特征保持效果。

### Q. MiniMax 图生图请求示例是什么?

下面是 MiniMax 图生图的请求体示例,使用 image-01 模型,通过 `subject_reference` 传入人物参考图,生成"女孩在图书馆窗前看向远方"的新图,同时保留参考图中的人物特征。

```python
payload = {
    "model": "image-01",
    "prompt": "女孩在图书馆的窗户前,看向远方",
    "aspect_ratio": "16:9",
    "subject_reference": [
        {
            "type": "character",
            "image_file": "https://example.com/reference.jpg"
        }
    ],
    "response_format": "url"
}
```

完整请求结构与文生图一致,需在 headers 中携带 `Authorization: Bearer <MINIMAX_API_KEY>`,POST 至 `https://api.minimaxi.com/v1/image_generation`。

## MiniMax 图像模型计费

### Q. MiniMax 图像模型如何计费?

MiniMax 图像模型按生成图像的张数计费,两款模型单价相同,与分辨率、宽高比无关。

- **image-01**:0.025 元 / 张
- **image-01-live**:0.025 元 / 张

计费按实际生成的图像数量(`n` 参数)累加。例如单次请求 `n=4`,则按 4 张计费,合计 0.1 元。
