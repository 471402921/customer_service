# 技术类报错排查

本文汇总 MiniMax 开放平台模型使用过程中常见的技术类报错与排查方法,涵盖通用错误码、余额与套餐错误、Claude Code 客户端报错、OpenRouter 第三方接入、网络与连接问题、MCP 工具配置、VSCode 插件配置以及 Token Plan 多模态使用限制等。

## 通用错误码

### Q. 错误码 1000 / 1305 / 520 / unknown error 怎么办?

**错误码 1000 / 1305 / 520 / unknown error** 通常由短时间内流量暴涨导致,线上呈现为 "1000 / 1305 / 520 / unknown error" 错误提示。

**排查与处理建议:**

1. 等待并稍后重试,平台正在进行紧急扩容和性能优化,服务将在短时间内逐步恢复稳定。
2. 若长时间持续报错,可避开高峰时段(工作日上午 11:00 左右、下午 14:00–18:00)再次尝试。
3. `1000 unknown error` 在个别情况下属于资源瞬时耗尽导致的异常错误,本质仍属于资源拥塞问题,稍后重试即可。

### Q. 错误码 529 / Rate Limited / 响应超时报错怎么办?

**错误码 529 / Rate Limited / 响应超时** 多出现在高峰时段,平台为保障整体系统稳定运行,会在高峰时段实施必要的流量调度与阶段性速率限制策略。

**当前主要的高峰时段:**

- 工作日上午 11:00 左右
- 工作日下午 14:00 – 18:00

**常见报错类型及含义:**

- **2062 / 2064**:当前处于高峰时段,Token Plan 的速率限制可能会临时收紧。建议稍后重试,并适当控制请求并发度。
- **1000 unknown error**:个别情况下的资源瞬时耗尽导致的异常错误,本质仍属于资源拥塞问题。
- **1002 rate limit exceeded (RPM)**:请求频率超过当前分钟级速率限制,请在下一分钟后重试。该错误不属于阶段性的速率限制。
- **2056 usage limit exceeded**:Token Plan 的累计请求量已达到套餐上限。该错误不属于阶段性的速率限制。

**用户侧建议:**

1. **分散定时任务**:尽量将定时任务分散到非整点时间(例如 10:17、10:42 等),避免在整点附近集中执行形成瞬时峰值,这将有效降低全局拥塞。
2. **合理使用资源**:平台会持续监测异常的高频调用行为,对于明显超出正常使用范畴的请求,将采取必要的限制措施,以保障绝大多数用户的正常使用体验。
3. **避开高峰时段**:在工作日上午 11:00 左右、下午 14:00–18:00 之外的时段调用,成功率更高。

**平台改进进展:**

- Status 页面预计将在 2 周内上线,方便实时查看系统运行状态与服务可用性。
- 工单系统标准化问题反馈与支持流程,预计将在 1 个月内上线。
- 平台正在实施更大规模的算力补充与调度优化措施,目标是在高峰时段将请求成功率从当前约 80% 提升至更高水平。

### Q. 错误码 1002 (rate limit exceeded RPM) 怎么解决?

**错误码 1002 rate limit exceeded (RPM)** 表示请求频率超过当前分钟级速率限制。

**排查步骤:**

1. 在下一分钟后重试请求。
2. 适当降低请求并发度,避免短时间内集中发起大量请求。
3. 该错误不属于阶段性的速率限制,而是常态化的 RPM 上限保护机制。

### Q. 错误码 2056 (usage limit exceeded) 怎么解决?

**错误码 2056 usage limit exceeded** 表示 Token Plan 的累计请求量已达到套餐上限。

**排查步骤:**

1. 检查当前 Token Plan 套餐的额度使用情况。
2. 等待套餐周期重置,或升级到更高级别的 Token Plan 套餐。
3. 若需要更高的额度或并发,可改用按量付费 API。

## 余额与套餐相关报错

### Q. insufficient balance (余额不足) 报错怎么办?

**insufficient balance(余额不足)报错**首先要排查 API Key 类型是否用错。

**排查步骤:**

1. **检查 API Key 类型是否匹配付费方式**:
   - Token Plan 订阅用户需使用 `sk-cp` 开头的 Key
   - 按量计费用户(即扣余额)需使用 `sk-api` 开头的 Key
2. **若 Key 类型没有用错,且账户没有欠费**,但模型仍响应 `insufficient balance`,请重置 API Key 后重新进行配置。

### Q. 错误码 2061 (套餐不支持该模型) 怎么解决?

**错误码 2061** 表示当前订阅的套餐不支持所调用的模型。

**Token Plan 支持的多模态模型清单:**

- M2.7
- music-2.5
- image-01
- speech-2.8
- MiniMax-Hailuo-2.3 / MiniMax-Hailuo-2.3-Fast

**排查步骤:**

1. 检查您调用的模型名称是否在套餐支持范围内。
2. 若需使用其他模型,可切换到按量付费方式。
3. Token Plan 完整可用模型列表见官网:https://platform.minimaxi.com/docs/token-plan/intro

### Q. 错误码 2062 (请求量过高) 怎么解决?

**错误码 2062** 表示当前请求量过高。Token Plan 主要面向个人开发者的交互式使用场景。

**排查步骤:**

1. 当前请求量较高,请稍后重试。
2. 如需支持更高并发或自动化任务,建议升级至更高级别套餐。
3. 也可改用按量付费 API,以获得更大的并发能力。

### Q. 错误码 2064 (服务集群负载过高) 怎么解决?

**错误码 2064** 表示当前服务集群负载较高。

**排查步骤:**

1. 请稍后重试,感谢您的耐心等待。
2. 高峰时段(工作日上午 11:00 左右、下午 14:00–18:00)更易出现 2064,可避开高峰时段调用。
3. 适当控制请求并发度,降低对集群的瞬时压力。

## Claude Code 客户端报错

### Q. Claude Code 出现 401 authentication_error 怎么办?

**Claude Code 401 authentication_error 报错**通常由 BaseURL 错误或 API Key 错误导致。

**排查步骤:**

1. **检查 BaseURL 是否正确**:
   - 国内服务填 `https://api.minimaxi.com/anthropic`,要特别注意 `minimax` 后面还有个 `i`(域名为 `minimaxi.com`,不是 `minimax.com`)
   - 海外服务填 `https://api.minimax.io/anthropic`
2. **检查 API Key 是否匹配付费方式**:
   - Token Plan 用户需使用 `sk-cp` 开头的 Key,需从 Token Plan 页面**重置后复制**获取,**不要直接复制带有 "..." 的 Key**(脱敏显示的 Key 无法使用)
   - 按量计费用户需使用 `sk-api` 开头的 Key

### Q. Claude Code 出现要求跳转到 Claude 官网登录怎么办?

**Claude Code 要求跳转到 Claude 官网登录**的解决方法是修改 `.claude.json` 配置文件。

**排查步骤:**

1. 打开 `.claude.json` 文件。
2. 在第一层大括号内查找是否有 `hasCompletedOnboarding` 字段:
   - 若**已存在**,将其值由 `false` 改为 `true`
   - 若**不存在**,在第一层大括号内新增配置:`"hasCompletedOnboarding": true`
3. 保存文件后重新启动 Claude Code。

## OpenRouter 接入相关

### Q. OpenRouter 上 TPS 速度偏低,和官网口径不一样怎么办?

**OpenRouter 上 TPS 偏低**并非模型原生性能问题,而是由聚合平台的统计口径与运行机制导致。

**TPS 偏低的主要原因:**

- **统计口径不同**:OpenRouter 按全请求生命周期统计,包含排队、网络传输、首包等待等耗时。
- **多用户混合调度**:聚合网关面向海量用户服务,存在请求排队和并发限流。
- **场景与负载差异**:平台展示为混合长度请求的整体均值。

**实测数据与建议:**

- 经验证,MiniMax M2.7-highspeed 稳态输出 TPS 稳定在 100 左右,符合官方标称。
- **建议直接使用官方 API 接口**(国内 `https://api.minimaxi.com`,海外 `https://api.minimax.io`)以获得最佳速度体验。

## 网络与连接问题

### Q. 网络环境导致连接失败或授权失败怎么办?

**网络环境导致的连接失败或授权失败**在某些场景下可能出现,常见原因包括公司网络代理、VPN 干扰、本地证书问题等。

**排查步骤:**

1. **公司网络通常有防火墙或代理设置**,即使用个人设备也可能受到影响,建议切换到手机热点或家庭网络进行排查。
2. **使用 VPN(梯子)可能导致连接异常**,建议关闭 VPN 后重试。
3. **若出现"网址授权失败"**,通常是网络环境或电脑网络证书问题导致,更换网络后重试。

## MCP 工具配置问题

### Q. MCP Server 配置了但工具无法正常调用怎么办?

**MCP Server 配置后工具无法调用**是常见问题,通常涉及依赖、BaseURL、API Key、连接状态等多个环节。

**排查步骤:**

1. **检查 uv 是否正确安装**:`uv` 是 MCP Server 运行的前置依赖,缺失会导致 MCP Server 无法启动。
2. **检查 MCP 配置中的 Base URL 是否正确**:国内服务为 `https://api.minimaxi.com`,注意 `minimax` 后面还有个 `i`(域名为 `minimaxi.com`)。
3. **在 Claude Code 中输入 `/mcp` 查看 MCP Server 连接状态**,确认已成功连接。
4. **确认配置文件中的 API Key 是否为最新的 `sk-cp` 开头的 Key**,需从 Token Plan 页面**重置后复制**获取。
5. **若连接状态正常但仍无法调用工具**,可在对话中明确提示模型使用指定工具(例如「请使用 xxx 工具完成 yyy」)。
6. **完整配置教程参考**:https://platform.minimaxi.com/docs/token-plan/mcp-guide

## VSCode 插件配置问题

### Q. Claude Code 的 VSCode 插件配置需要注意什么?

**Claude Code CLI 和 VSCode 的 Claude 插件使用不同的配置文件,不能混用**。这是配置不一致导致一边可用一边报错的最常见原因。

**配置文件区别:**

- **Claude Code CLI 的 MCP 配置**位于全局配置文件 `~/.claude.json` 或项目级 `.claude/settings.json` 中。
- **VSCode 的 Claude 插件**有自己独立的配置文件,其中的 MCP 配置**不会被 Claude Code CLI 读取**。

**排查步骤:**

1. 如果同时使用 Claude Code CLI 和 VSCode 插件,需分别检查两边的配置(Base URL、API Key、模型名称等)是否保持一致,避免因配置不统一导致一边可用一边报错。
2. 在 VSCode 插件中也要确认 Base URL 正确(国内 `https://api.minimaxi.com/anthropic`,海外 `https://api.minimax.io/anthropic`)。
3. 在 VSCode 插件中也要确认 API Key 类型匹配付费方式(Token Plan 用 `sk-cp`,按量计费用 `sk-api`)。
4. 可通过 `/mcp` 命令验证当前环境下的 MCP 连接状态。

## Token Plan 多模态使用限制

### Q. Token Plan 如何使用图像识别? 图像识别有误怎么办?

**Token Plan 图像识别**支持两种接入方式,均需通过命令行或 MCP 调用。

**接入方式:**

- **方式一:接入 Token Plan MCP**,配置教程参考 https://platform.minimaxi.com/docs/token-plan/mcp-guide
- **方式二:安装 MiniMax CLI 工具箱**(https://github.com/MiniMax-AI/cli),使用 `mmx vision <图片路径>` 命令进行识别。

**使用注意事项:**

- 需提供图片的**文件路径**,目前不支持直接截图粘贴到命令行的方式。
- 若识别有误,可尝试使用更清晰的图片重新识别,或在 prompt 中提供更明确的引导。

### Q. Token Plan 如何使用图像生成、语音生成、视频生成、音乐生成?

**Token Plan 多模态生成能力**(图像生成、语音生成、视频生成、音乐生成)需配置对应的工具。

**接入方式:**

- **方式一:安装 MiniMax 官方提供的 Skill —— minimax-multimodal-toolkit**,地址:https://github.com/MiniMax-AI/skills/tree/main/skills/minimax-multimodal-toolkit
- **方式二:安装 MiniMax CLI 工具箱**(https://github.com/MiniMax-AI/cli),通过命令行调用各类生成能力。

### Q. Token Plan 有语音转文字服务吗?

**Token Plan 目前未直接提供语音转文字(ASR)服务**,可通过安装第三方 Skill 来实现。

**推荐方案:**

- 安装 **Whisper ASR Skill**,这是开源免费的语音识别 Skill,安装后即可在对话中进行音频/语音转文字。

### Q. Token Plan 使用 highspeed 模型提示不支持怎么办?

**Token Plan 使用 highspeed 模型提示不支持**,是因为 `highspeed` 后缀的模型有套餐等级限制。

**说明:**

- `highspeed` 后缀的模型(如 `M2.7-highspeed`)**仅限极速版(Turbo)套餐用户使用**。
- 若当前不是 Turbo 套餐用户,需升级到极速版套餐才能调用 highspeed 模型。

### Q. Token Plan 调用语音/图片/视频/音乐生成时显示不支持为什么?

**Token Plan 调用语音/图片/视频/音乐生成时显示不支持**,通常是因为调用了 Token Plan 不支持的旧版模型。

**排查步骤:**

1. 查看 Token Plan 可用的模型列表:https://platform.minimaxi.com/docs/token-plan/intro
2. **旧版模型(如 `speech-02-turbo`)Token Plan 不支持调用**。
3. 若有使用旧版模型的需求,可通过**按量计费**的方式充值使用。
