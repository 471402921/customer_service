# 音乐模型使用指南

本指南介绍 MiniMax 开放平台音乐模型的能力矩阵、调用方式、参数说明、请求示例与计费规则。MiniMax 音乐模型当前提供三类能力:歌曲生成、翻唱生成、歌词生成,分别对应不同接口与计费规则。

接口与文档入口:

- 音乐生成接口文档:https://platform.minimaxi.com/docs/api-reference/music-generation
- 音乐生成使用指南:https://platform.minimaxi.com/docs/guides/music-generation
- 歌词生成接口文档:https://platform.minimaxi.com/docs/api-reference/lyrics-generation

## 模型矩阵

### Q. MiniMax 音乐模型有哪些版本,分别适合哪些创作场景?

MiniMax 开放平台提供 4 款音乐模型,覆盖歌曲生成与翻唱两大场景,每类能力分付费版与免费版:

- **music-2.6**(推荐):最新音乐生成模型,以声传情,翻唱入心、器乐入魂。支持歌曲生成与纯音乐生成。仅限 Token Plan 用户和付费用户使用,RPM 更高。
- **music-cover**:翻唱生成模型,基于参考音频一键生成翻唱版本,支持风格迁移和自动歌词提取。仅限 Token Plan 用户和付费用户使用,RPM 更高。
- **music-2.6-free**:music-2.6 的免费版本,所有用户可通过 API Key 使用,RPM 较低。
- **music-cover-free**:music-cover 的免费版本,所有用户可通过 API Key 使用,RPM 较低。

### Q. Music 2.6 相比前代音乐模型有哪些核心提升?

Music 2.6 在能力维度、表现力、结构精度、声音设计上均有显著升级:

- **翻唱能力**:新增 Music Cover 模型,支持基于参考音频生成翻唱版本。
- **纯音乐生成**:通过 `is_instrumental` 参数支持纯音乐/器乐生成。
- **自动歌词生成**:通过 `lyrics_optimizer` 参数可从 prompt 自动生成歌词。
- **流式输出**:支持 stream 模式实时返回音频数据。
- **编曲混音**:扩展高采样率音色库(含管弦、民乐等),优化声场算法。
- **人声表现**:优化 AI 合成音痕迹,引入拟人化音色模拟,Flow 表现力增强。
- **结构精度**:支持 Intro/Verse/Chorus/Bridge/Interlude/Build-up/Hook 等 14 种以上结构标签。
- **声音设计**:自动识别并还原目标风格的关键物理特性。

## 歌曲生成(music-2.6 / music-2.6-free)

### Q. MiniMax 音乐生成接口怎么调用,通用参数有哪些?

请求端点:`POST https://api.minimaxi.com/v1/music_generation`

通用参数(适用于所有音乐生成模型):

- **model**(必填):music-2.6 / music-cover / music-2.6-free / music-cover-free。
- **prompt**(条件必填):音乐风格、情绪、场景描述,最长 2000 字符。music-2.6 纯音乐模式下必填;music-cover 模式下必填(描述目标翻唱风格,10-300 字符)。
- **lyrics**(条件必填):歌词内容,可含结构标签,最长 3500 字符。music-2.6 非纯音乐模式下必填(1-3500 字符);music-cover 模式下选填(不填则自动从参考音频提取,10-1000 字符)。
- **stream**(选填):是否开启流式输出,默认 false。
- **output_format**(选填):返回方式,url(24 小时有效)或 hex,默认 hex。stream=true 时仅支持 hex。
- **audio_setting.sample_rate**(选填):采样率,可选 16000 / 24000 / 32000 / 44100。
- **audio_setting.bitrate**(选填):比特率,可选 32000 / 64000 / 128000 / 256000。
- **audio_setting.format**(选填):输出格式,mp3 / wav / pcm。
- **aigc_watermark**(选填):是否在音频末尾添加水印,默认 false,仅非流式模式有效。

### Q. music-2.6 / music-2.6-free 有哪些专属参数?

music-2.6 与 music-2.6-free 在通用参数基础上,新增两个能力开关:

- **lyrics_optimizer**(选填):是否根据 prompt 自动生成歌词,默认 false。开启后无需自行填写 `lyrics`,模型会基于 `prompt` 自动生成歌词内容。
- **is_instrumental**(选填):是否生成纯音乐(无人声),默认 false。开启后进入纯音乐模式,此时 `prompt` 必填,`lyrics` 可不填。

### Q. music-2.6 歌曲生成的请求示例是什么?

以下是一个 music-2.6 生成独立民谣风格歌曲的 Python 调用示例,包含 prompt 与含结构标签的 lyrics:

```python
import requests, os

url = "https://api.minimaxi.com/v1/music_generation"
headers = {
    "Content-Type": "application/json",
    "Authorization": f"Bearer {os.environ.get('MINIMAX_API_KEY')}"
}

payload = {
    "model": "music-2.6",
    "prompt": "独立民谣,忧郁,内省,渴望,独自漫步,咖啡馆",
    "lyrics": "[verse]\n街灯微亮晚风轻抚\n影子拉长独自漫步\n[chorus]\n推开木门香气弥漫\n熟悉的角落陌生人看",
    "audio_setting": {
        "sample_rate": 44100,
        "bitrate": 256000,
        "format": "mp3"
    }
}

response = requests.post(url, headers=headers, json=payload)
```

### Q. 音乐生成支持哪些歌词结构标签?

MiniMax 音乐生成支持 14 种歌词结构标签,可在 `lyrics` 字段中使用,用于精确控制歌曲段落结构:

- `[Intro]`:前奏
- `[Verse]`:主歌
- `[Pre Chorus]`:导歌(副歌前过渡)
- `[Chorus]`:副歌
- `[Interlude]`:间奏
- `[Bridge]`:桥段
- `[Outro]`:尾声
- `[Post Chorus]`:副歌后段
- `[Transition]`:转场
- `[Break]`:停顿/休止
- `[Hook]`:钩子段
- `[Build Up]`:铺垫递进
- `[Inst]`:器乐段
- `[Solo]`:独奏

## 翻唱生成(music-cover / music-cover-free)

### Q. music-cover 翻唱生成模型怎么使用?

Music Cover 模型可基于一段参考音频生成翻唱版本,支持风格迁移和自动歌词提取。

使用步骤:

1. 准备参考音频:时长 6 秒至 6 分钟,大小不超过 50 MB,支持 mp3 / wav / flac 等常见音频格式。
2. 调用接口时提供参考音频(URL 或 Base64)和目标翻唱风格描述(`prompt`)。
3. 可选提供歌词(`lyrics`);不提供时系统自动从参考音频中提取。

请求端点同音乐生成接口:`POST https://api.minimaxi.com/v1/music_generation`。

### Q. music-cover / music-cover-free 有哪些专属参数?

music-cover 与 music-cover-free 在通用参数基础上,需要传入参考音频(URL 或 Base64 二选一):

- **audio_url**(条件必填):参考音频 URL,与 audio_base64 二选一。
- **audio_base64**(条件必填):参考音频的 Base64 编码,与 audio_url 二选一。

参考音频要求:时长 6 秒至 6 分钟,大小不超过 50 MB,支持 mp3 / wav / flac 等常见音频格式。

此外,music-cover 模式下:

- `prompt` 必填,描述目标翻唱风格,长度 10-300 字符。
- `lyrics` 选填,若不填则系统自动从参考音频中提取歌词,长度 10-1000 字符。

### Q. music-cover 翻唱生成的请求示例是什么?

以下示例展示如何用 music-cover 把一段参考音频翻唱为"轻快流行女声"风格(歌词由模型自动从参考音频提取):

```python
payload = {
    "model": "music-cover",
    "prompt": "轻快的流行风格翻唱,女声演唱",
    "audio_url": "https://example.com/reference_song.mp3",
    "audio_setting": {
        "sample_rate": 44100,
        "bitrate": 256000,
        "format": "mp3"
    }
}

response = requests.post(url, headers=headers, json=payload)
```

## 歌词生成

### Q. MiniMax 歌词生成接口如何使用,关键参数有哪些?

歌词生成是独立接口,可用于完整创作或在已有歌词基础上续写/改写,产物可直接用于音乐生成接口的 `lyrics` 字段。

接口文档:https://platform.minimaxi.com/docs/api-reference/lyrics-generation
请求端点:`POST https://api.minimaxi.com/v1/lyrics_generation`

参数:

- **mode**(必填):`write_full_song`(完整歌曲创作)或 `edit`(编辑/续写)。
- **prompt**(必填):歌词创作提示,最长 2000 字符。留空则随机生成。
- **lyrics**(选填):已有歌词,仅 `edit` 模式有效,最长 3500 字符。
- **title**(选填):歌曲标题,提供后输出会保留该标题。

返回结果包含:

- **song_title**:歌曲名。
- **style_tags**:风格标签。
- **lyrics**:含结构标签的歌词,可直接用于音乐生成接口的 `lyrics` 参数。

### Q. 歌词生成的调用示例是什么?

以下是一个用 `write_full_song` 模式生成"欢乐新年歌曲"的 Python 调用示例:

```python
payload = {
    "mode": "write_full_song",
    "prompt": "一首欢乐的新年歌曲",
    "title": "新年快乐"
}

response = requests.post(
    "https://api.minimaxi.com/v1/lyrics_generation",
    headers=headers, json=payload
)
```

## 计费规则

### Q. MiniMax 音乐模型如何计费?

MiniMax 音乐模型按"首"或"次"计费,具体如下:

- **Music-2.6**:1.0 元/首(限时免费推广中,为期 2 周)。
- **Music-2.5+**:1.0 元/首。
- **Music-2.5**:1.0 元/首。
- **Music-2.0**:0.25 元/首。
- **歌词生成**:0.05 元/次(限时免费推广中,为期 2 周)。

Token Plan 用户额外福利:所有套餐均可使用 Music-2.6,每日额度 100 首(每首 ≤ 5 分钟),限时免费推广期间不扣额度。
