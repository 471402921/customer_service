# MiniMax 文件管理使用指南

## 文件管理概述

### Q. MiniMax 文件管理接口能做什么?

MiniMax 开放平台文件管理接口提供 5 个能力:文件上传、按用途列出文件、按 file_id 检索文件信息、下载文件二进制内容、删除文件。

通用信息:

- Base URL:`https://api.minimaxi.com`
- 认证方式:Bearer Token(API Key)
- 接口文档总入口:https://platform.minimaxi.com/docs/api-reference/file-management-upload

## 文件上传接口

### Q. MiniMax 文件上传接口怎么用?支持哪些用途和格式?

MiniMax 文件上传接口用于把本地文件上传至开放平台,供后续语音克隆、提示音频、异步语音合成等场景使用。

接口信息:

- 接口文档:https://platform.minimaxi.com/docs/api-reference/file-management-upload
- 端点:`POST /v1/files/upload`
- Content-Type:`multipart/form-data`

请求参数:

- `purpose`(必填):文件用途,可选值为 `voice_clone` / `prompt_audio` / `t2a_async_input`
- `file`(必填):本地文件路径

各 purpose 支持的文件格式:

- `voice_clone`(声音克隆音频):支持 mp3、m4a、wav
- `prompt_audio`(提示音频):支持 mp3、m4a、wav
- `t2a_async_input`(异步文本转语音输入):支持 text、zip

调用示例(curl):

```bash
curl --location 'https://api.minimaxi.com/v1/files/upload' \
  --header 'Authorization: Bearer <API_KEY>' \
  --form 'purpose="voice_clone"' \
  --form 'file=@"/path/to/sample.mp3"'
```

## 文件列出接口

### Q. 如何按用途列出 MiniMax 上已经上传的文件?

MiniMax 文件列出接口用于按 `purpose` 分类,查询当前账号下已上传的全部文件清单。

接口信息:

- 接口文档:https://platform.minimaxi.com/docs/api-reference/file-management-list
- 端点:`GET /v1/files/list?purpose={purpose}`

请求参数:

- `purpose`(必填):文件用途,需与上传时使用的 `purpose` 一致

调用示例(curl):

```bash
curl --location 'https://api.minimaxi.com/v1/files/list?purpose=voice_clone' \
  --header 'Authorization: Bearer <API_KEY>'
```

## 文件检索接口

### Q. 如何通过 file_id 检索 MiniMax 文件信息与下载地址?

MiniMax 文件检索接口用于通过 `file_id` 查询单个文件的元信息,返回内容包含 `download_url` 下载地址。

接口信息:

- 接口文档:https://platform.minimaxi.com/docs/api-reference/file-management-retrieve
- 端点:`GET /v1/files/retrieve?file_id={file_id}`

请求参数:

- `file_id`(必填):文件唯一标识符

支持检索以下场景返回的 `file_id`:

- 视频生成任务状态查询返回的 `file_id`
- 异步语音合成任务状态查询返回的 `file_id`

返回信息中包含 `download_url`,其中视频文件下载 URL 的有效期为 1 小时,过期后需重新调用本接口重新获取。

调用示例(curl):

```bash
curl --location 'https://api.minimaxi.com/v1/files/retrieve?file_id=xxxx' \
  --header 'Authorization: Bearer <API_KEY>'
```

## 文件下载接口

### Q. 如何直接下载 MiniMax 文件二进制内容?

MiniMax 文件下载接口用于通过 `file_id` 直接拉取文件原始二进制内容,适用于不需要先获取 `download_url` 而希望直接落盘的场景。

接口信息:

- 接口文档:https://platform.minimaxi.com/docs/api-reference/file-management-retrieve-content
- 端点:`GET /v1/files/retrieve_content?file_id={file_id}`

请求参数:

- `file_id`(必填):文件唯一标识符

调用示例(curl):

```bash
curl --location 'https://api.minimaxi.com/v1/files/retrieve_content?file_id=xxxx' \
  --header 'Authorization: Bearer <API_KEY>' \
  --output downloaded_file
```

## 文件删除接口

### Q. MiniMax 文件删除接口如何调用?

MiniMax 文件删除接口用于按 `file_id` 和 `purpose` 删除已上传的文件。

接口信息:

- 接口文档:https://platform.minimaxi.com/docs/api-reference/file-management-delete
- 端点:`POST /v1/files/delete`

请求参数:

- `file_id`(必填):文件唯一标识符
- `purpose`(必填):文件用途,可选值为 `voice_clone` / `prompt_audio` / `t2a_async` / `t2a_async_input` / `video_generation`

调用示例(curl):

```bash
curl --location 'https://api.minimaxi.com/v1/files/delete' \
  --header 'Authorization: Bearer <API_KEY>' \
  --header 'Content-Type: application/json' \
  --data '{"file_id":"xxxx","purpose":"voice_clone"}'
```

## 错误码说明

### Q. MiniMax 文件管理接口的常见错误码分别代表什么?

调用 MiniMax 文件管理接口时,响应 `base_resp.status_code` 字段会返回错误码。常见错误码及含义如下:

- `0`:请求成功
- `1000`:未知错误
- `1001`:超时
- `1002`:触发 RPM 限流(每分钟请求次数超限)
- `1004`:鉴权失败(API Key 无效或未携带)
- `1008`:余额不足
- `1013`:服务内部错误
- `1026`:输入内容涉及敏感内容
- `1027`:输出内容涉及敏感内容
- `1039`:触发 TPM 限流(每分钟 Token 数超限)
- `2013`:传入参数异常(请检查请求参数格式是否正确)
- `2049`:无效的 API Key

完整错误码列表参见:https://platform.minimaxi.com/docs/api-reference/errorcode
