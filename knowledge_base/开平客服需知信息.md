# 开平客服需知信息

## 官网与接口地址

MiniMax 开放平台分国内和海外两个版本：

- **国内开放平台**：https://platform.minimaxi.com
- **海外开放平台**：https://www.minimax.io/platform（仅支持谷歌邮箱登录，建议使用方便登录谷歌账号的邮箱注册）

### API 请求接口地址（Base URL）

调用 OpenAI 兼容格式接口时，Base URL 如下：

- **国内用户**：`https://api.minimaxi.com/v1`
- **国际用户**：`https://api.minimax.io/v1`

调用 Anthropic 兼容格式接口时，Base URL 如下：

- **国内版**：`https://api.minimaxi.com/anthropic`（注意域名为 minimaxi.com）
- **国外版**：`https://api.minimax.io/anthropic`（注意域名为 minimax.io）

> 注意区分国内域名（minimaxi.com）与国际域名（minimax.io），两者不可混用。

#### 发票抬头或类型有误时的重开流程

若 API 发票的抬头或发票类型填写有误，需走发票重开流程：

1. 提供原票进行**红冲**
2. 再开具新票

普通发票换开增值税专用发票，需额外提供以下信息：

- 开户银行
- 银行账号
- 公司地址
- 公司电话
- 资质证明文件（三选一上传）：
  - 历史开具增值税专票的图片
  - 税局系统查询显示"一般纳税人"状态的截图
  - 一般增值税纳税人证书
- 上传委托证明：《增值税专用发票申请委托及承诺书》（PDF 格式，可联系客服获取模板）

### Coding Plan 开票

Coding Plan 订单开票入口：https://platform.minimaxi.com/user-center/payment/coding-plan

支持开票的支付方式：

| 支付方式 | 是否可开票 |
|----------|------------|
| 支付宝全额支付 | 可以开票 |
| 开放平台余额支付 | 可以开票 |
| 开放平台余额 + 支付宝组合支付 | 可以开票 |
| 代金券抵扣部分 | **不可开票** |

> 若订单中使用了代金券，开票金额为扣除代金券后的实际支付金额。

若 Coding Plan 开票界面显示"暂无可开票订单"，属于正常情况，本月内会支持此功能，无需担心。

### 其他产品开票说明

除 API 开票需要一定周期外，其他产品的开票均为**自助开票，随买随开**。若长时间未收到发票，常见原因为开票申请页面填写有误或邮箱等开票信息填写有误。

## 账户注册与企业认证

MiniMax 开放平台目前支持**手机号**完成账户注册。注册后可绑定邮箱等其他方式，便于多样化认证和账号找回。平台实行**一号一户**原则，不建议手机号换绑。

### 企业认证流程

企业认证入口：https://platform.minimaxi.com/user-center/basic-information

认证步骤：

1. 校验认证信息
2. 填写绑定账户信息（银行账户）
3. 填写鉴权金额（银行打款金额）
4. 完成认证

企业认证注意事项：

- **企业认证银行与打款银行信息必须一致**
- 认证完成后**不支持换绑**银行账户
- 若出现报错 `该待绑定账户已绑定在其他会员子账户上(1016)`，说明该银行卡已被其他账户绑定，建议更换银行卡重新绑定

### 账户注销

如需注销账户，请发送姓名、Group ID、注册手机号至邮箱 nashi@minimaxi.com。**账户注销后不得再次注册使用**。

## 接口报错排查

不同接口类型报错时，需要提供不同的 ID 才能定位问题：

| 接口类型 | 需要提供的 ID |
|----------|--------------|
| 视频接口 | Task ID |
| 语音接口 | Trace ID |
| 音乐接口 | Trace ID |
| M 系列模型 | Request ID（类似 Trace ID） |

获取 Trace ID 的方式：

```python
print("Trace-ID:", response.headers['Trace-ID'])
```

## 代金券

MiniMax 开放平台代金券分为两种类型，适用范围不同。

**试用代金券**：在开放平台完成个人认证或企业认证后自动获得。可用于开放平台中所有模型的 API 调用消耗抵扣（优先抵扣），**不能**用于购买资源包（包括 Coding Plan）的抵扣。

**激励代金券**：通过 https://platform.minimaxi.com/user-center/basic-information 页面中的「Coding Plan 共建邀请计划」获得。可用于开放平台中所有模型的 API 调用消耗抵扣（优先抵扣），**可以**用于购买资源包（包括 Coding Plan）的抵扣。

| 代金券类型 | 获取方式 | 模型调用抵扣 | 购买资源包/Coding Plan |
|------------|----------|-------------|----------------------|
| 试用代金券 | 完成个人/企业认证后获得 | 支持 | **不支持** |
| 激励代金券 | Coding Plan 共建邀请计划 | 支持 | **支持** |

> 使用代金券支付的部分不可开具发票，发票仅针对实际支付金额开具。
