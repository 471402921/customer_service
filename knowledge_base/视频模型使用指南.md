# 视频模型使用指南

## MiniMax 视频模型有哪些，分别支持哪些生成能力？

MiniMax 视频生成模型系列为 **MiniMax-Hailuo**，当前主力版本是 **MiniMax-Hailuo-02**。

支持的视频生成模式：

- **文生视频（T2V）**：根据文本描述直接生成视频，所有版本支持
- **图生视频（I2V）**：基于起始图片 + 文本描述生成动态视频，所有版本支持
- **首尾帧视频生成（FL2V）**：指定首帧和尾帧图片，自动补全中间过渡，仅 MiniMax-Hailuo-02 支持
- **主体参考视频生成（S2V）**：通过人脸参考图 + 文本生成保持人物面部一致性的视频，所有版本支持
- **视频 Agent（模板生成）**：将图片或文本素材填充到预设模板中，快速生成风格统一的视频

---

## 文生视频（T2V）

### 文生视频怎么调用，关键参数有哪些？

文生视频根据文本描述直接生成视频。

接口文档：https://platform.minimaxi.com/docs/api-reference/video-generation-t2v

**请求端点：**
```
POST https://api.minimaxi.com/v1/video_generation
```

**关键参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| model | string | 模型名称，如 `MiniMax-Hailuo-02` |
| prompt | string | 视频内容描述，支持运镜指令 |
| duration | int | 视频时长（秒） |
| resolution | string | 分辨率，如 `1080P`、`720P`、`512P` |

文生视频支持通过运镜指令控制镜头，在 prompt 中添加如 `[左移]`、`[推进]`、`[跟随]` 等指令。运镜指令详见「视频 Prompt 写作指南」章节。

---

## 图生视频（I2V）

### 图生视频如何使用，对输入图片有什么要求？

图生视频基于起始图片，结合文本描述生成动态视频。

接口文档：https://platform.minimaxi.com/docs/api-reference/video-generation-i2v

**图片要求：**
- 格式：JPG / JPEG / PNG / WebP
- 大小：< 20MB
- 短边：> 300px
- 长宽比：2:5 ~ 5:2（即不能太宽或太窄）

**请求端点：**
```
POST https://api.minimaxi.com/v1/video_generation
```

传入 `first_frame_image` 参数（Base64 编码或 URL）指定起始图片，配合 `prompt` 描述视频动态内容。

---

## 首尾帧视频生成（FL2V）

### 首尾帧视频生成适合什么场景，使用时要注意什么？

首尾帧模式适合"起止画面已知、过渡过程自动补全"的场景，例如：物体形变、场景切换、人物动作连接等。

接口文档：https://platform.minimaxi.com/docs/api-reference/video-generation-fl2v

**使用限制：**
- 仅支持 `MiniMax-Hailuo-02` 模型
- 不支持 512P 分辨率
- 输出视频尺寸遵循首帧图片的宽高比

**使用方式：** 同时传入 `first_frame_image`（首帧）和 `last_frame_image`（尾帧），模型自动生成中间过渡内容。

---

## 主体参考视频生成（S2V）

### 主体参考视频生成怎么保证人物一致性？

主体参考模式通过人脸参考图 + 文本描述生成视频，重点是保持人物面部一致性，适合需要固定角色形象的内容创作场景。

接口文档：https://platform.minimaxi.com/docs/api-reference/video-generation-s2v

**使用方式：** 传入 `subject_reference`（人脸参考图，Base64 或 URL），配合 `prompt` 描述视频内容，模型在生成过程中保持人脸与参考图一致。

**图片建议：** 使用清晰正面人脸照片，避免遮挡或侧脸，以提升一致性效果。

## 视频 Agent（模板生成）

### 视频 Agent 怎么用？

视频 Agent 可将图片或文本素材填充到预设模板中，快速生成风格统一的视频，适合批量内容生产或固定风格输出。

**相关接口文档：**
- 创建任务：https://platform.minimaxi.com/docs/api-reference/video-agent-create
- 查询状态：https://platform.minimaxi.com/docs/api-reference/video-agent-query
- 模板列表：https://platform.minimaxi.com/docs/faq/video-agent-templates

**请求端点：**
```
POST https://api.minimaxi.com/v1/video_template_generation
```

使用流程：
1. 从模板列表选择合适的模板，获取 `template_id`
2. 调用创建接口，传入 `template_id` 和所需素材
3. 轮询查询接口获取任务状态（参考通用工作流程）

视频 Agent 任务成功后，下载 URL 有效期为 **9 小时**（普通视频生成为 1 小时）。

## 视频生成通用工作流程

### 视频生成任务的通用工作流程是什么？

所有视频生成接口均采用**异步任务**模式，工作流程如下：

**第一步：创建生成任务**

调用相应的生成接口，成功后返回 `task_id`。

**第二步：轮询查询任务状态**

通过 `task_id` 定期查询任务进度，状态流转：

```
Preparing → Queueing → Processing → Success / Fail
```

- `Preparing`：任务初始化
- `Queueing`：排队等待
- `Processing`：生成中
- `Success`：生成成功，可获取下载 URL
- `Fail`：生成失败

建议轮询间隔：3~5 秒。

**第三步：下载视频**

任务状态为 `Success` 后，从响应中获取视频下载 URL：
- 普通视频生成：URL 有效期约 **1 小时**
- 视频 Agent：URL 有效期约 **9 小时**

请在有效期内下载，过期需重新查询或重新生成。

## 视频 Prompt 写作指南

### 写视频 Prompt 时怎么提升可控性与出片质量？

详细指南：https://platform.minimaxi.com/docs/guides/video-prompt

**推荐 Prompt 结构：**

```
主体 + 动作 + 场景 + 风格 + 运镜
```

示例：`一只金毛犬在海滩上奔跑，阳光明媚，写实风格，[跟随]`

**支持的运镜指令（15 种）：**

| 类型 | 指令 |
|------|------|
| 左右移动 | `[左移]`、`[右移]` |
| 左右摇镜 | `[左摇]`、`[右摇]` |
| 推拉镜头 | `[推进]`、`[拉远]` |
| 升降镜头 | `[上升]`、`[下降]` |
| 上下摇镜 | `[上摇]`、`[下摇]` |
| 变焦 | `[变焦推近]`、`[变焦拉远]` |
| 其他 | `[晃动]`、`[跟随]`、`[固定]` |

**运镜组合规则：**
- 同一 `[ ]` 内的多个指令**同时生效**，建议不超过 3 个，例如：`[推进 上升 左摇]`
- 前后出现的多个 `[ ]` 指令**按顺序生效**，例如：`[推进] 然后 [拉远]`

**写作建议：**
- 主体描述要具体（物种/人物特征/颜色等）
- 动作描述用动词短语，避免静态描述
- 风格词可参考：写实、动漫、油画、赛博朋克、电影感等
- 运镜指令放在 prompt 末尾或对应描述片段之后
