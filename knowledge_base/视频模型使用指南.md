# 视频模型使用指南

本文档介绍 MiniMax 开放平台视频模型的型号矩阵、五种生成模式(文生视频、图生视频、首尾帧、主体参考、视频 Agent 模板)、输入图片要求、视频生成任务通用工作流程,以及视频 prompt 撰写技巧。

## 视频模型型号矩阵

### Q. MiniMax 视频模型有哪些型号,各自支持哪些生成能力和分辨率?

MiniMax 开放平台目前提供 4 款视频模型,各模型支持的生成模式与分辨率不同。

- **MiniMax-Hailuo-2.3**:支持文生视频、图生视频;分辨率 768P(6 秒/10 秒)、1080P(6 秒)。
- **MiniMax-Hailuo-2.3-Fast**:支持图生视频;分辨率 768P(6 秒/10 秒)、1080P(6 秒)。
- **MiniMax-Hailuo-02**:支持文生视频、图生视频、首尾帧、主体参考;分辨率 512P/768P/1080P。
- **S2V-01**:支持主体参考生成视频;分辨率 720P(6 秒)。

选型建议:文生视频或图生视频优先选 MiniMax-Hailuo-2.3 或 MiniMax-Hailuo-02;追求图生视频生成速度选 MiniMax-Hailuo-2.3-Fast;首尾帧场景必须用 MiniMax-Hailuo-02;主体参考(人物一致性)场景用 S2V-01 或 MiniMax-Hailuo-02。

## 文生视频(Text-to-Video)

### Q. 文生视频适用什么场景,怎么调用,有哪些关键参数?

**适用场景**:仅有文本描述、无参考图片时,根据文字直接生成视频。例如分镜脚本可视化、广告 demo、概念演示。

**接口文档**:https://platform.minimaxi.com/docs/api-reference/video-generation-t2v

**请求端点**:`POST https://api.minimaxi.com/v1/video_generation`

**关键参数**:

- **model**(必填):MiniMax-Hailuo-2.3 或 MiniMax-Hailuo-02。
- **prompt**(必填):视频文本描述,最大 2000 字符。
- **duration**(选填):视频时长(秒),默认 6,可选 6 或 10。
- **resolution**(选填):分辨率,768P(默认)或 1080P。
- **prompt_optimizer**(选填):是否自动优化 prompt,默认 true。
- **callback_url**(选填):接收任务状态更新的回调 URL。

**限制与提示**:文生视频支持通过运镜指令控制镜头(如 `[左移]`、`[推进]`、`[跟随]`),具体运镜词见本文档「视频 prompt 撰写技巧」章节。

## 图生视频(Image-to-Video)

### Q. 图生视频适用什么场景,怎么调用,有哪些关键参数?

**适用场景**:已有起始画面(首帧),想让画面动起来。例如让一张产品照片自动旋转展示、让人物照片做特定动作。

**接口文档**:https://platform.minimaxi.com/docs/api-reference/video-generation-i2v

**请求端点**:`POST https://api.minimaxi.com/v1/video_generation`

**关键参数**:

- **model**(必填):MiniMax-Hailuo-2.3、MiniMax-Hailuo-2.3-Fast 或 MiniMax-Hailuo-02。
- **first_frame_image**(必填):起始帧图片,支持 URL 或 Base64。
- **prompt**(选填):视频文本描述,用于控制运动方向与内容。

**限制与提示**:输入图片必须满足本文档「输入图片要求」章节列出的格式、大小、尺寸条件,否则任务会失败。生成的视频尺寸遵循首帧图片宽高比。

## 首尾帧视频(First-Last Frame to Video)

### Q. 首尾帧视频生成适用什么场景,怎么调用,有哪些限制?

**适用场景**:起止画面已知、过渡过程需要自动补全的视频。例如做镜头切换、转场动画、A 状态到 B 状态的过渡演示。

**接口文档**:https://platform.minimaxi.com/docs/api-reference/video-generation-fl2v

**请求端点**:`POST https://api.minimaxi.com/v1/video_generation`

**关键参数**:

- **model**(必填):MiniMax-Hailuo-02(仅此模型支持首尾帧)。
- **last_frame_image**(必填):结束帧图片,URL 或 Base64。
- **first_frame_image**(选填):起始帧图片,URL 或 Base64。

**限制与提示**:

- 首尾帧模式仅支持 MiniMax-Hailuo-02 模型,其他模型不可用。
- 不支持 512P 分辨率。
- 视频尺寸遵循首帧图片宽高比。
- 输入图片需满足本文档「输入图片要求」章节的限制。

## 主体参考视频(Subject Reference to Video)

### Q. 主体参考视频生成怎么保证人物一致性,有哪些关键参数?

**适用场景**:需要在视频中保持人物面部一致的场景。例如让指定人物完成多种动作、生成同一角色的系列短片。

**接口文档**:https://platform.minimaxi.com/docs/api-reference/video-generation-s2v

**请求端点**:`POST https://api.minimaxi.com/v1/video_generation`

**关键参数**:

- **model**(必填):S2V-01。
- **subject_reference**(必填):主体参考数组,`type=character`,单张人脸参考图片。
- **prompt**(选填):视频文本描述,用于控制动作、场景、风格。

**限制与提示**:S2V-01 仅支持主体参考模式,固定输出 720P / 6 秒。参考图片需为人脸,建议使用清晰正面照,避免遮挡或侧脸,以提升一致性效果。图片同样需满足本文档「输入图片要求」章节的格式与尺寸限制。

## 视频 Agent(模板生成)

### Q. 视频 Agent 模板生成是什么,什么场景下使用,怎么调用?

**适用场景**:将图片或文本素材填充到预设模板中,快速生成风格统一的视频。适合批量生产同款风格内容(如商品短视频、节日祝福、口播片头)。

**接口文档**:

- 创建任务:https://platform.minimaxi.com/docs/api-reference/video-agent-create
- 查询状态:https://platform.minimaxi.com/docs/api-reference/video-agent-query
- 模板列表:https://platform.minimaxi.com/docs/faq/video-agent-templates

**请求端点**:`POST https://api.minimaxi.com/v1/video_template_generation`

**使用流程**:

1. 从模板列表选择合适的模板,获取 `template_id`。
2. 调用创建接口,传入 `template_id` 和所需素材。
3. 轮询查询接口获取任务状态(参考本文档「视频生成任务通用工作流程」章节)。

**限制与提示**:视频 Agent 模板生成的下载 URL 有效期为 9 小时(普通视频生成为 1 小时),需注意区分。具体可用模板见上述模板列表链接。

## 输入图片要求

### Q. 视频模型对输入图片有哪些统一要求(格式/大小/尺寸)?

凡使用图生视频、首尾帧、主体参考模式时,输入图片(包括 first_frame_image、last_frame_image、subject_reference)必须同时满足以下条件:

- **格式**:JPG、JPEG、PNG、WebP。
- **文件大小**:小于 20MB。
- **短边像素**:大于 300 像素。
- **长宽比**:在 2:5 到 5:2 之间(即不能太宽或太窄)。

不满足上述任一条件,任务会被拒绝或失败。建议提交前先在本地校验图片参数。

## 视频生成任务通用工作流程

### Q. 调用视频生成接口后,怎么获取最终视频?

视频生成是异步任务,所有视频接口统一遵循以下三步流程:

1. **创建生成任务**:调用对应接口,成功后返回 `task_id`。
2. **轮询查询任务状态**:用 `task_id` 定期查询任务进度,状态流转为 `Preparing` → `Queueing` → `Processing` → `Success` / `Fail`。建议轮询间隔 3 至 5 秒。
3. **下载视频**:任务状态为 `Success` 后,通过返回的下载 URL 获取视频文件。

**任务状态含义**:

- `Preparing`:任务初始化。
- `Queueing`:排队等待。
- `Processing`:生成中。
- `Success`:生成成功,可获取下载 URL。
- `Fail`:生成失败。

**下载 URL 有效期**:

- 普通视频生成(文生/图生/首尾帧/主体参考):约 1 小时。
- 视频 Agent 模板生成:约 9 小时。

建议任务成功后立即下载并转存到自有存储,避免链接过期。

## 视频 prompt 撰写技巧

视频 prompt 直接影响生成质量与可控性,本节按子主题分别说明推荐结构、运镜指令清单与组合规则。

**详细指南**:https://platform.minimaxi.com/docs/guides/video-prompt

### Q. 视频 prompt 推荐用什么结构来组织?

推荐按 **主体 + 动作 + 场景 + 风格 + 运镜** 五要素顺序撰写 prompt:

- **主体**:画面中的核心对象,例如「一只白色小猫」「身穿红色连衣裙的女孩」「一只金毛犬」。
- **动作**:主体在做什么,用动词短语描述,避免静态描述。例如「在海滩上奔跑」「转身回眸」。
- **场景**:环境与时间,例如「黄昏的海边」「下雪的街道」「阳光明媚的草地」。
- **风格**:画面质感与艺术风格,例如「电影感」「日系动漫风」「写实纪录片」「油画」「赛博朋克」。
- **运镜**:镜头运动方式,使用方括号标注的运镜指令,通常放在 prompt 末尾或对应描述片段之后。

示例:`一只金毛犬在海滩上奔跑,阳光明媚,写实风格,[跟随]`。

要素描述越具体、越完整,生成效果越可控。

### Q. 视频模型支持哪些运镜指令?

视频模型共支持 15 种运镜指令,统一使用方括号 `[ ]` 包裹,可分为以下几类:

- **左右移**:`[左移]`、`[右移]`(镜头水平平移)。
- **左右摇**:`[左摇]`、`[右摇]`(镜头水平转动)。
- **推拉**:`[推进]`、`[拉远]`(镜头前后移动)。
- **升降**:`[上升]`、`[下降]`(镜头垂直移动)。
- **上下摇**:`[上摇]`、`[下摇]`(镜头垂直转动)。
- **变焦**:`[变焦推近]`、`[变焦拉远]`(焦距变化,镜头位置不变)。
- **其他**:`[晃动]`、`[跟随]`、`[固定]`。

注意区分「移」(镜头位置移动)与「摇」(镜头位置不变,角度转动)的差异。

### Q. 多个运镜指令怎么组合使用?

运镜指令支持组合,组合规则如下:

- **同一括号内多指令并行**:同一个 `[ ]` 中放多个指令时同时生效,例如 `[推进 上升 左摇]`。建议同时生效的指令不超过 3 个,避免画面不可控。
- **多个括号顺序生效**:prompt 中前后出现多个 `[ ]` 时,按出现顺序在视频时间轴上依次生效,例如「主体描述 [推进] 然后 [拉远]」。

合理组合运镜可模拟电影分镜效果,但过度堆叠会让画面抖动或失去主体感,建议先单指令测试再组合。
