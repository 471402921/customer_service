# 文本模型使用指南

本指南介绍 MiniMax 开放平台（国内域名 `minimaxi.com`）文本模型的型号矩阵、API 调用方式（Anthropic 兼容、OpenAI 兼容、专用对话接口）、核心特性（Tool Use、交错思维链、Prompt 缓存）以及常见疑问。

## 文本模型矩阵与适用场景

### Q. MiniMax 文本模型有哪些型号，分别适合什么场景？

MiniMax 开放平台提供以下文本模型，M2 系列上下文窗口均为 204800 token，并同时支持 Anthropic API 与 OpenAI API 两种调用方式；M2-her 走专用文本对话接口。

- **MiniMax-M2.7**：开启模型自我迭代，输出速度约 60 TPS，上下文 204800，支持 Anthropic API / OpenAI API。适合复杂推理、Tool Use、需要交错思维链的任务。
- **MiniMax-M2.7-highspeed**：M2.7 极速版，效果不变、更快更敏捷，输出速度约 100 TPS，上下文 204800，支持 Anthropic API / OpenAI API。适合延迟敏感、大并发的生产环境。
- **MiniMax-M2.5**：顶尖性能与极致性价比，输出速度约 60 TPS，上下文 204800，支持 Anthropic API / OpenAI API。适合日常通用任务、批量处理。
- **MiniMax-M2.5-highspeed**：M2.5 极速版，效果不变速度提升，输出速度约 100 TPS，上下文 204800，支持 Anthropic API / OpenAI API。适合高吞吐场景。
- **MiniMax-M2.1**：强大多语言编程能力，输出速度约 60 TPS，上下文 204800,支持 Anthropic API / OpenAI API。适合多语言代码生成、代码工程。
- **MiniMax-M2.1-highspeed**：M2.1 极速版，效果不变速度提升，输出速度约 100 TPS，上下文 204800，支持 Anthropic API / OpenAI API。适合实时编程辅助。
- **MiniMax-M2**：专为高效编码与 Agent 工作流而生，上下文 204800，支持 Anthropic API / OpenAI API。适合 Agentic 任务编排、自动化工作流。
- **M2-her**：角色扮演、多轮对话场景专用模型，使用 MiniMax 文本对话专用接口（非 Anthropic / OpenAI 兼容路径）。

## 文本模型 API 调用方式

### Q. MiniMax 文本模型 API 如何调用？

MiniMax 文本模型兼容 **Anthropic SDK（推荐）** 和 **OpenAI SDK** 两种调用方式。国内开放平台使用 `minimaxi.com` 域名，两种模式 Base URL 路径不同：Anthropic 兼容用 `/anthropic`，OpenAI 兼容用 `/v1`，请勿混用。

**Anthropic API（推荐）调用**

接口文档：https://platform.minimaxi.com/docs/api-reference/text-anthropic-api

Base URL：`https://api.minimaxi.com/anthropic`

```plaintext
import anthropic

client = anthropic.Anthropic(
    base_url="https://api.minimaxi.com/anthropic",
    api_key="YOUR_API_KEY"
)

message = client.messages.create(
    model="MiniMax-M2.7",
    max_tokens=1000,
    system="You are a helpful assistant.",
    messages=[{"role": "user", "content": "Hi, how are you?"}]
)

```

Anthropic 兼容模式支持特性：流式输出、Interleaved Thinking（交错思维链）、Tool Use、Prompt 缓存。

**OpenAI API 兼容模式调用**

接口文档：https://platform.minimaxi.com/docs/api-reference/text-openai-api

Base URL：`https://api.minimaxi.com/v1`

```plaintext
from openai import OpenAI

client = OpenAI(
    base_url="https://api.minimaxi.com/v1",
    api_key="YOUR_API_KEY"
)

response = client.chat.completions.create(
    model="MiniMax-M2.7",
    messages=[{"role": "user", "content": "Hi, how are you?"}]
)

```

OpenAI 兼容模式支持特性：流式输出、Interleaved Thinking（通过 `extra_body={"reasoning_split": True}` 启用）、Tool Use。

### Q. M2-her 模型应该如何调用？

M2-her 使用 MiniMax 专用文本对话接口（不走 Anthropic / OpenAI 兼容路径），支持 system、user_system、group 等角色设定和示例对话学习，更适合角色扮演与多轮对话场景。

接口文档：https://platform.minimaxi.com/docs/api-reference/text-chat

### Q. 使用 MiniMax 文本模型时，环境变量应该怎么配置？

MiniMax 文本模型环境变量需根据调用方式分别配置，国内开放平台 Base URL 使用 `minimaxi.com` 域名，注意 Anthropic 兼容模式路径为 `/anthropic`、OpenAI 兼容模式路径为 `/v1`。

```plaintext
# Anthropic API（推荐）
export ANTHROPIC_BASE_URL=https://api.minimaxi.com/anthropic
export ANTHROPIC_API_KEY=YOUR_API_KEY

# OpenAI API
export OPENAI_BASE_URL=https://api.minimaxi.com/v1
export OPENAI_API_KEY=YOUR_API_KEY

```

## 文本模型核心特性

### Q. MiniMax 文本模型有哪些核心特性？

MiniMax 文本模型在编程、办公、推理效率、运行成本四个方向均有显著优势：

- **编程**：像架构师一样思考和构建，支持 10+ 编程语言。
- **办公**：Word、PPT、Excel 金融建模等场景可交付。
- **效率**：100 TPS 推理速度，SWE-Bench 平均耗时降至 22.8 分钟（提速 37%）。
- **成本**：100 TPS 下每小时连续运行约 1 美金。

### Q. Tool Use 与交错思维链在 MiniMax 文本模型中怎么正确使用？

MiniMax-M2.7 原生支持 Interleaved Thinking（交错思维链），能够在每轮 Tool Use 前进行思考并决定下一步动作。

使用指南：https://platform.minimaxi.com/docs/guides/text-m2-function-call

**关键注意事项**

- 多轮 Function Call 对话中，**必须把完整 assistant 返回加入对话历史**。
- Anthropic SDK：`response.content` 包含 thinking / text / tool_use 等全部内容块，需要完整回传。
- OpenAI SDK：使用 `reasoning_split=True` 获取独立思考字段，同样需要完整保留。

### Q. MiniMax 文本模型 Prompt 缓存如何帮助降低成本和延迟？

使用 Prompt 缓存可以同时降低延迟与成本；缓存读取价格约为输入价格的 1/10，适合长 system prompt 或固定上下文反复使用的场景。

接口文档：https://platform.minimaxi.com/docs/api-reference/text-prompt-caching

### Q. MiniMax 文本模型 Anthropic 主动缓存如何配置？

Anthropic 兼容模式支持通过显式设置来管理 cache，可实现更细粒度的缓存控制，适合需要精确控制缓存命中的场景。

接口文档：https://platform.minimaxi.com/docs/api-reference/anthropic-api-compatible-cache

## 文本模型常见疑问

### Q. MiniMax-M2.1、MiniMax-M2.1-highspeed、MiniMax-M2.7、MiniMax-M2.7-highspeed 模型是否支持图像识别？

MiniMax-M2 系列模型本身并不具备图像识别能力。Token Plan（原 Coding Plan）用户可通过配置 Token Plan MCP 来实现图片识别和网络检索的功能。

Token Plan MCP 具体配置方法，请参考文档：https://platform.minimaxi.com/docs/coding-plan/mcp-guide

### Q. MiniMax-M2.7-highspeed 模型比 MiniMax-M2.7 快多少？

MiniMax-M2.7 的推理输出速度约 60 TPS，MiniMax-M2.7-highspeed 在效果不变的基础上速度提升至约 100 TPS。highspeed 版本适合对延迟敏感、需要高吞吐量的生产环境。

### Q. 为什么在第三方平台（如 OpenRouter）上看到的 MiniMax 文本模型 TPS 偏低？

OpenRouter 等聚合平台上显示的 TPS 偏低，并非 MiniMax 文本模型原生性能问题，主要原因如下：

- **统计口径不同**：OpenRouter 按全请求生命周期统计，包含排队、网络传输、首包等待等耗时。
- **多用户混合调度**：聚合网关面向海量用户服务，存在请求排队和并发限流。
- **场景与负载差异**：平台展示为混合长度请求的整体均值。

经验证，MiniMax-M2.7-highspeed 稳态输出 TPS 稳定在 100 左右，符合官方标称。建议直接使用 MiniMax 官方 API 接口（`https://api.minimaxi.com/anthropic` 或 `https://api.minimaxi.com/v1`）以获得最佳速度体验。
