# MiniMax 文本模型使用指南

## MiniMax 文本模型有哪些，分别适合什么场景？

MiniMax 文本模型系列包括 M2 系列和 M2-her，覆盖通用对话、编程、Agent 工作流、角色扮演等场景。

**MiniMax-M2.7**
- 定位：旗舰推理模型，开启模型自我迭代
- 输出速度约 60 TPS，上下文窗口 204800 tokens
- 适用场景：复杂推理、代码编写、工具调用（Tool Use）、需要交错思维链（Interleaved Thinking）的任务

**MiniMax-M2.7-highspeed**
- 定位：M2.7 极速版，效果与 M2.7 一致
- 输出速度约 100 TPS，上下文窗口 204800 tokens
- 适用场景：对响应速度要求高、大并发的生产环境

**MiniMax-M2.5**
- 定位：顶尖性能与极致性价比，轻松驾驭复杂任务
- 输出速度约 60 TPS，上下文窗口 204800 tokens
- 适用场景：日常通用任务、需要高性价比的批量处理

**MiniMax-M2.5-highspeed**
- 定位：M2.5 极速版，效果与 M2.5 一致
- 输出速度约 100 TPS，上下文窗口 204800 tokens
- 适用场景：高吞吐量、延迟敏感型应用

**MiniMax-M2.1**
- 定位：强大多语言编程能力，全面升级代码工程体验
- 输出速度约 60 TPS，上下文窗口 204800 tokens
- 适用场景：多语言代码生成、代码工程、编程助手

**MiniMax-M2.1-highspeed**
- 定位：M2.1 极速版，效果与 M2.1 一致
- 输出速度约 100 TPS，上下文窗口 204800 tokens
- 适用场景：实时编程辅助、大并发代码生成

**MiniMax-M2**
- 定位：专为高效编码与 Agent 工作流而生
- 上下文窗口 204800 tokens
- 适用场景：Agentic 任务编排、自动化工作流

**M2-her**
- 定位：文本对话模型，专为角色扮演和多轮对话设计
- 使用专用文本对话接口，支持 system、user_system、group 等角色设定和示例对话学习
- 适用场景：角色扮演、虚拟陪伴、多轮情感对话

## MiniMax 文本模型 API 如何调用？

MiniMax 文本模型兼容 **Anthropic SDK**（推荐）和 **OpenAI SDK** 两种调用方式。

### Anthropic API 调用（推荐）

接口文档：https://platform.minimaxi.com/docs/api-reference/text-anthropic-api

**Base URL：** `https://api.minimaxi.com/anthropic`

```python
import anthropic

client = anthropic.Anthropic(
    base_url="https://api.minimaxi.com/anthropic",
    api_key="YOUR_API_KEY"
)

message = client.messages.create(
    model="MiniMax-M2.7",
    max_tokens=1000,
    system="You are a helpful assistant.",
    messages=[{"role": "user", "content": "Hi, how are you?"}]
)
```

Anthropic API 支持特性：流式输出、Interleaved Thinking（交错思维链）、Tool Use、Prompt 缓存。

### OpenAI API 兼容模式调用

接口文档：https://platform.minimaxi.com/docs/api-reference/text-openai-api

**Base URL：** `https://api.minimaxi.com/v1`

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.minimaxi.com/v1",
    api_key="YOUR_API_KEY"
)

response = client.chat.completions.create(
    model="MiniMax-M2.7",
    messages=[{"role": "user", "content": "Hi, how are you?"}]
)
```

OpenAI API 支持特性：流式输出、Interleaved Thinking（通过 `extra_body={"reasoning_split": True}` 启用）、Tool Use。

## M2-her 应该如何调用？

M2-her 使用专用文本对话接口，与其他 M2 系列调用方式不同。

接口文档：https://platform.minimaxi.com/docs/api-reference/text-chat

M2-her 支持 system、user_system、group 等角色设定，以及示例对话学习功能，更适合角色扮演与多轮对话场景。

## MiniMax 文本模型有哪些核心特性？

- **编程**：像架构师一样思考和构建，支持 10+ 编程语言
- **办公**：Word、PPT、Excel 金融建模场景可交付
- **效率**：100 TPS 推理速度，SWE-Bench 平均耗时降至 22.8 分钟（37% 提速）
- **成本**：100 TPS 下每小时连续运行约 1 美金

---

## Tool Use 与交错思维链（Interleaved Thinking）如何使用？

使用指南：https://platform.minimaxi.com/docs/guides/text-m2-function-call

MiniMax-M2.7 原生支持 Interleaved Thinking，能够在每轮 Tool Use 前进行思考并决定下一步动作。

**关键注意事项（多轮 Function Call 必读）**

多轮 Function Call 对话中，**必须把完整 assistant 返回加入对话历史**：

- **Anthropic SDK**：`response.content` 包含 thinking/text/tool_use 等全部内容块，需完整回传
- **OpenAI SDK**：使用 `reasoning_split=True` 获取独立思考字段，同样需要完整保留

## Prompt 缓存如何帮助降低成本和延迟？

接口文档：https://platform.minimaxi.com/docs/api-reference/text-prompt-caching

使用 Prompt 缓存可以同时降低延迟与成本：
- 缓存读取价格约为输入价格的 **1/10**
- 适合长 system prompt 或固定上下文反复使用的场景

## 使用 MiniMax 文本模型时，环境变量应该怎么配置？

```bash
# Anthropic API（推荐）
export ANTHROPIC_BASE_URL=https://api.minimaxi.com/anthropic
export ANTHROPIC_API_KEY=YOUR_API_KEY

# OpenAI API
export OPENAI_BASE_URL=https://api.minimaxi.com/v1
export OPENAI_API_KEY=YOUR_API_KEY
```

## MiniMax-M2 系列模型是否支持图像识别？

MiniMax-M2.1、MiniMax-M2.1-highspeed、MiniMax-M2.7、MiniMax-M2.7-highspeed 模型本身**不具备图像识别能力**。

Coding Plan 用户可通过配置 Coding Plan MCP 来实现图片识别和网络检索功能。

Coding Plan MCP 配置方法，请参考文档：https://platform.minimaxi.com/docs/coding-plan/mcp-guide

---

## MiniMax-M2.7-highspeed 比 MiniMax-M2.7 快多少？

- MiniMax-M2.7 推理输出速度约 **60 TPS**
- MiniMax-M2.7-highspeed 在效果不变的基础上，速度提升至约 **100 TPS**

highspeed 版本适合对延迟敏感、需要高吞吐量的生产环境。
