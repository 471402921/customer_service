# MiniMax 快速入门与使用指南

本文档介绍 MiniMax 开放平台的接入准备、API Key 类型选择、企业团队权限管理、MCP 工具体系、API 能力总览，以及 Token Plan 多模态能力上手方式。Token Plan 套餐的详细计费规则与权益请参见《Token_Plan专项.md》。

## 接入准备

### Q. 接入 MiniMax 开放平台之前需要做哪些准备？

接入 MiniMax 开放平台前需要完成三步：账户注册、创建 API Key、按需充值。

详细文档：https://platform.minimaxi.com/docs/guides/quickstart-preparation

**第一步：注册 MiniMax 开放平台账户**

注册入口：https://platform.minimaxi.com/login

**第二步：创建并保存可用的 API Key**

API Key 管理入口：https://platform.minimaxi.com/user-center/basic-information/interface-key

根据使用场景选择 Key 类型：

- 以 `sk-cp` 开头的 Key 用于 Token Plan 订阅套餐，在 https://platform.minimaxi.com/user-center/payment/token-plan 点击「重置并复制」获取，仅可调用文本模型
- 以 `sk-api` 开头的 Key 用于按量付费，在 https://platform.minimaxi.com/user-center/basic-information/interface-key 创建，可调用文本、语音、视频、音乐、图像等全模态接口

**第三步：按量调用前先完成充值**

如果选择按量付费方式调用 API，需要先在账户充值后再发起调用。

充值入口：https://platform.minimaxi.com/user-center/payment/balance

## API Key 类型与选型

MiniMax 开放平台共有三类 API Key / 计费形式：普通 API Key（按量付费）、Token Plan Key（订阅套餐）、Coding Plan（编程订阅）。三者面向不同场景，本节分别说明。

### Q. MiniMax 普通 API Key（sk-api）适用什么场景？

普通 API Key 是以 `sk-api` 开头的密钥，用于按量付费调用 MiniMax 全模态 API。

**适用场景**：

- 调用文本、语音、视频、音乐、图像等任意模态接口
- 业务量可大可小、希望按实际用量结算（按 token / 字符 / 次计费）
- 在自己的应用、后端服务、SDK 中集成 MiniMax 能力

**创建入口**：https://platform.minimaxi.com/user-center/basic-information/interface-key

**前置条件**：使用前需要先在 https://platform.minimaxi.com/user-center/payment/balance 完成账户充值。

### Q. MiniMax Token Plan Key（sk-cp）适用什么场景？

Token Plan Key 是以 `sk-cp` 开头的密钥，用于 Token Plan 订阅套餐，按 prompt 数量计费而非 token 数。

**适用场景**：

- 在 Claude Code、Cursor、OpenClaw 等 AI 编程工具中使用 MiniMax 文本模型
- 希望按订阅套餐方式（固定包内 prompt 数量）使用文本模型，控制成本
- 仅需调用文本模型，不需要语音/视频/图像/音乐生成

**重要限制**：Token Plan Key 仅支持文本模型，不能用于多模态接口。如需多模态生成能力，参见本文「Token Plan 用户如何使用多模态能力」一节。

**获取方式**：在 https://platform.minimaxi.com/user-center/payment/token-plan 点击「重置并复制」获取。

Token Plan 详细的套餐规格、prompt 计费规则、配额限制等请参见《Token_Plan专项.md》。

### Q. MiniMax Coding Plan 是什么？

Coding Plan 是 MiniMax 面向编程场景的订阅形态，专门针对在 AI 编程工具中调用文本模型的高频场景做了优化。它与 Token Plan 同属订阅类计费，与按量付费的普通 API Key 路径不同。

Coding Plan 的具体套餐规格、配额、计费细则与对应权益请参见《Token_Plan专项.md》中的对应章节。

### Q. MiniMax API Key 选哪种？普通 API Key、Token Plan Key、Coding Plan 怎么选？

按场景一句话决策：

- 在 Claude Code、Cursor、OpenClaw 等 AI 编程工具里只用文本模型 → 优先选 **Token Plan Key**（`sk-cp` 开头）或 **Coding Plan**
- 需要调用语音、视频、图像、音乐等多模态生成能力 → 必须使用 **普通 API Key**（`sk-api` 开头），按量付费
- 自建应用 / 后端服务 / SDK 集成、用量不固定 → **普通 API Key**，按量付费

如果同时有编程辅助和多模态需求，可以同时持有两类 Key 分别用于不同场景。

## 企业团队权限管理

### Q. MiniMax 开放平台企业团队如何管理账号与权限？

企业团队建议采用「主账号 + 子账号」方式管理 MiniMax 开放平台的账号与权限，便于团队协作时控制支付权限、共享配额。

**主账号 + 子账号机制要点**：

- 子账号默认没有「支付」相关的查看与管理权限，由主账号统一负责充值与计费
- 子账号与主账号共享平台权益与速率限制（rate limit），不会因为多子账号而额外提高配额
- 多个子账号可以共用主账号下的余额、Token Plan 配额等资源

**子账号管理入口**：https://platform.minimaxi.com/user-center/basic-information/child-account

## MiniMax MCP（模型上下文协议）

MiniMax 提供两套 MCP：面向通用多模态调用的 **MiniMax MCP**，以及面向 Token Plan 用户的 **Token Plan MCP**。两者定位不同，分别说明。

### Q. MiniMax MCP 是什么？有哪些官方实现？

MiniMax MCP 是 MiniMax 提供的官方模型上下文协议（MCP, Model Context Protocol）工具，用于在支持 MCP 的 AI Agent / IDE 中直接调用 MiniMax 的多模态能力。

详细文档：https://platform.minimaxi.com/docs/guides/mcp-guide

**官方实现**：

- Python 版本：https://github.com/MiniMax-AI/MiniMax-MCP
- JavaScript 版本：https://github.com/MiniMax-AI/MiniMax-MCP-JS

### Q. MiniMax MCP 内置了哪些常用工具？

MiniMax MCP 内置以下常用工具，覆盖语音、音乐、视频、图像四个模态：

- `text_to_audio`：文本合成语音
- `list_voices`：查询可用音色
- `voice_clone`：通过音频文件克隆音色
- `voice_design`：通过提示词生成音色
- `play_audio`：播放音频文件
- `music_generation`：通过提示词与歌词生成音乐
- `generate_video`：文本或图片生成视频
- `image_to_video`：首帧图像生成视频（JavaScript 版本）
- `query_video_generation`：查询视频生成任务状态
- `text_to_image`：通过提示词生成图片

### Q. Token Plan MCP 是什么？提供了哪些专属能力？

Token Plan MCP 是 MiniMax 专门为 Token Plan 用户提供的 MCP 工具，与通用的 MiniMax MCP 不同，它聚焦于编程场景下的辅助能力。

详细文档：https://platform.minimaxi.com/docs/guides/token-plan-mcp-guide

**Token Plan MCP 专属工具**：

1. **网络搜索**：在编码过程中快速检索网络信息
2. **图片理解**：识别并理解图片内容

### Q. MiniMax MCP 与 Token Plan MCP 有什么区别？

MiniMax MCP 与 Token Plan MCP 是两套不同的 MCP 工具，定位与适用对象不同：

- **MiniMax MCP**：通用多模态 MCP，提供语音合成、音色克隆/设计、音乐生成、视频生成、图像生成等多模态工具，使用普通 API Key（`sk-api`），按量付费
- **Token Plan MCP**：Token Plan 用户专属，提供网络搜索、图片理解两项编程辅助工具，使用 Token Plan Key（`sk-cp`）

如果你是 Token Plan 用户但需要多模态生成能力，应改用 MiniMax CLI（详见下文「Token Plan 多模态能力」一节），或使用普通 API Key 配 MiniMax MCP。

## MiniMax 开放平台 API 能力总览

### Q. MiniMax 开放平台都有哪些 API 能力？

MiniMax 开放平台 API 覆盖文本、语音、视频、图像、音乐、文件管理六大类。

API 概览文档：https://platform.minimaxi.com/docs/api-reference/api-overview

**文本类 API**：

- Anthropic 兼容接口
- OpenAI 兼容接口
- 文本对话
- Prompt 缓存

**语音类 API**：

- 同步合成（HTTP / WebSocket）
- 异步合成
- 音色复刻
- 音色设计
- 音色管理

**视频类 API**：

- 文生视频
- 图生视频
- 首尾帧
- 主体参考
- 视频 Agent

**图像类 API**：

- 文生图
- 图生图

**音乐类 API**：

- 音乐生成
- 歌词生成

**文件管理 API**：

- 上传
- 检索
- 列出
- 下载
- 删除

各模态的详细调用方式参见对应文档：《文本模型使用指南.md》《语音模型使用指南.md》《视频模型使用指南.md》《图像模型使用指南.md》《音乐模型使用指南.md》《文件管理使用指南.md》。

## Token Plan 多模态能力（MiniMax CLI）

### Q. Token Plan 用户如何使用多模态能力（视频/语音/音乐/图像生成）？

Token Plan 用户可以通过安装 **MiniMax CLI（mmx-cli）** 工具，在 AI Agent 中直接调用多模态能力，无需写代码。

**GitHub 仓库**：https://github.com/MiniMax-AI/cli

**配置文档**：https://platform.minimaxi.com/docs/token-plan/minimax-cli

**快速安装方式（推荐）**：

在 OpenClaw、Claude Code、Cursor 等 AI Agent 中发送如下指令，Agent 会自动完成安装：

```
帮我安装 MiniMax CLI：https://github.com/MiniMax-AI/cli
```

**手动安装方式**：

1. 全局安装：`npm install -g mmx-cli`
2. 完成认证：`mmx auth login --api-key sk-xxxxx`

安装并完成认证后，Token Plan 用户即可通过自然语言或 CLI 命令使用以下全部能力：视频生成、语音合成、音乐创作、图像生成、图像理解、网络搜索。
