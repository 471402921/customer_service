# 客服知识库梳理任务指令

## 背景

为 MiniMax 开放平台搭建基于七陌（7Moor）客服平台的智能客服 Agent。七陌底层基于 RAG，需要将现有飞书文档整理为 RAG 友好的 MD 文件导入知识库。

## 执行规则：一次会话处理一个链接

### 第一步：读取本次指定的飞书文档

从下方「任务链接列表」中取出本次会话要处理的链接，执行：

```bash
lark-cli docs +fetch --doc "<飞书文档URL>" --format pretty
```

> lark-cli 已安装并完成登录（用户：Jet.D），直接可用。

### 第二步：梳理输出 MD 文件

- 输出目录：`/Users/jet.d/minimax_work/Maas_ops/customer_service/knowledge_base/`
- 文件命名：根据飞书文档标题或核心主题命名，如 `开平客服需知信息.md`
- **一个飞书文档对应一个 MD 文件**，不按主题拆分为多个文件
- 文档内多个主题用 `##` 二级标题区分，在同一文件内组织

## RAG 友好文档规范

### 结构

- **一个文件一个主题** — 不同业务模块单独成文
- **标题用 markdown 语法** — `#` `##` `###`，不手写"1.1""3.2"等编号
- **段落控制在 2048 字符以内** — 七陌切片上限，超长按子话题拆分
- **列表表达层级** — 用有序/无序列表，不手写编号

### 内容

- **段落自包含** — 每段切出来后能独立理解，关键词重复出现，不用"它""该功能"等指代
- **答案直给** — 先结论后解释
- **关键词前置** — 按用户可能的问法组织语言
- **URL/链接原样保留** — 平台地址、表单链接完整保留
- **术语统一** — 同一概念全文一致表达
- **图片转文字** — 七陌无法解析图片语义，图片内容须改为文字描述
- **表格简化** — 复杂合并单元格改写为自然语言；保留简单表格时确保每行数据完整独立
